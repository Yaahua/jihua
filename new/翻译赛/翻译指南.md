### “优译杯”技术翻译说明与要求 (Translation Guidelines)

**1. 核心目标 (Overarching Goal)**

本次翻译的核心目标是产出一份**高度专业、精准且符合英文技术文档阅读习惯**的译文。您不仅仅是在翻译字面意思，更是在用英语为目标读者（海外技术人员、工程师）重新“撰写”这份技术文档。

**2. 核心原则 (Core Principles)**

您的所有翻译工作都应围绕以下四个评审维度展开：

*   **准确性 (Accuracy):** 概念、术语、数据绝不能出错。这是基石。
*   **可用性 (Usability):** 译文必须流畅、地道、易于理解，符合技术写作范式。
*   **规范性 (Specification):** 遵循格式、拼写、标点等硬性规定。
*   **创新性 (Innovation):** 在保证准确的前提下，进行“译写优化”，提升可读性，而非机械转换。

**3. 表达方式 (Expression Style)**

*   **语调 (Tone):** **专业、客观、自信 (Professional, Objective, Confident)**。避免口语化、情绪化的表达。
*   **句式 (Sentence Structure):**
    *   **多用短句：** 将中文长句拆分为多个逻辑清晰的英文短句。
    *   **多用被动语态：** 强调客观事实，而非操作主体。例如，用 "The system can be configured..." 而不是 "You can configure the system..."。
    *   **多用名词化结构：** 使语言更书面、更凝练。例如，用 "Upon detection of..." 替代 "When the system detects..."。
*   **用词 (Wording):**
    *   **精准：** 使用行业标准术语。例如，用 `associated loads` 替代 `related loads`。
    *   **规范：** 严格区分情态动词。**`Shall`** 用于强制性技术规范；**`Will`** 用于描述系统行为或结果；**`May`** 用于表示可能性或许可。
    *   **一致：** 同一术语在全文中必须保持统一译法。强烈建议创建并维护一个术语表。

**4. 思考方式 (Thinking Methodology)**

1.  **理解而非翻译 (Comprehend, then Write):** 在动手前，首先要完全理解中文原文的技术逻辑和核心信息。
2.  **读者优先 (Audience First):** 想象您的读者是一位毫无中文背景的外国工程师，他需要依靠您的文档去理解和操作。您的译文能让他毫不费力地看懂吗？
3.  **结构重组 (Restructure):** 不要被中文的句子结构束缚。在理解原文后，用最符合英文技术写作逻辑的方式重新组织句子和段落。
4.  **查证与验证 (Verify):** 对于不确定的术语，交叉参考您提供的术语表、Jeemaa 官网以及权威技术词典。

**5. 关键术语与背景 (Key Terminology & Context)**

*   **公司/平台:** 优译信息 (UEdrive), 译马网 (Jeemaa)
*   **产品/服务:**
    *   Jeemaa EE (企业版), Jeemaa FE (个人版)
    *   在线翻译平台 (online translation platform)
    *   计算机辅助翻译 (CAT - Computer-Assisted Translation)
    *   翻译记忆库 (Translation Memory / TM), 术语库 (Termbase / TB)
    *   游戏/软件/网站本地化 (Game/Software/Website Localization)
*   **技术理念 (From official website):**
    *   科技驱动翻译 (technology-driven translation)
    *   大模型翻译 (large model translation)
    *   私有化部署 (private deployment)

**6. 格式与提交 (Format & Submission)**

*   **匿名原则：** 最终提交的 `PDF` 译文和 `PPT` 说明中，**严禁**出现任何个人或学校信息。
*   **缩写规范：** 任何缩写在首次出现时，必须写出全称并后跟括号内的缩写。例如：`Message Queue (MQ)`。
